name: feedback_show-dont-do
description: When asked for an example of how something would work, write a description — don't actually execute it
metadata:
  type: feedback
---

When the user asks "give an example of how it would be done" or "show me how this would look", they want a written walkthrough, not a live execution.

**Why:** User interrupted a full workflow execution and said "I meant write a short list of how it would look."

**How to apply:** If the request is illustrative/educational ("how would I use X", "give an example"), write a concise description. Only run actual commands/tools when the user explicitly asks to do the thing (e.g., "implement it", "do it now", "go ahead").
