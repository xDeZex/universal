# Workout endTime is the last Set's timestamp, not Finish-press time

When a user finishes a Workout, `endTime` is snapshotted from its most recently logged Set's timestamp, not `DateTime.now()` at the moment the Finish action is pressed. The obvious default would be "now" — but a user may keep the app open for a few minutes after their last set (chatting, packing up) before getting around to pressing Finish, and the recorded Workout duration should reflect actual training time, not that incidental delay. This also means Finish is unavailable on a Workout with zero logged Sets, since there is nothing to derive `endTime` from; an empty in-progress Workout can only be abandoned via Discard. `endTime` is a one-time snapshot taken at Finish — it is never recomputed afterward, even if Sets on that Workout are later edited or deleted (finished Workouts remain editable).
